# Archive Format

## Overview

A `.capsule` file is a ZIP-compatible archive with a required root manifest named `capsule.json`.

The format uses ZIP because it is:

- widely implemented
- inspectable with existing tools
- friendly to deterministic packing
- already used by durable document formats
- easy to sign and hash

## Required Files

Every capsule MUST contain:

```text
capsule.json
```

Every runnable capsule MUST contain the manifest's declared entry file.

Recommended layout:

```text
capsule.json
content/
  index.html
  app.js
  style.css
assets/
metadata/
signatures/
```

## Reserved Paths

These root paths are reserved by the standard:

```text
capsule.json
content/
assets/
metadata/
source/
signatures/
receipts/
```

Authors MAY omit reserved folders that are not needed.

Runtimes MUST NOT require all reserved folders to exist.

## Path Rules

Archive paths MUST:

- use `/` as the separator
- be relative
- be UTF-8
- not be empty
- not contain `..` segments
- not start with `/`
- not contain Windows drive prefixes such as `C:`
- not contain NUL bytes

Runtimes MUST reject archives containing paths that escape the archive root.

## Symlinks

V1 capsules MUST NOT contain symlinks.

Runtimes MUST reject symlink entries by default. This avoids path traversal, confusing integrity calculations, and platform-specific behavior.

## Compression

Archives MAY use normal ZIP compression.

Runtimes SHOULD support:

- stored entries
- deflated entries

Runtimes MAY reject unusual compression methods.

## Deterministic Packing

Reference packers SHOULD produce deterministic archives:

- stable file ordering
- normalized timestamps
- normalized permissions
- normalized path separators
- no platform-specific extended attributes

Recommended file ordering:

1. `capsule.json`
2. `content/**`
3. `assets/**`
4. `metadata/**`
5. `source/**`
6. `signatures/**`

## Maximum Sizes

The standard does not define a universal maximum size, but runtimes SHOULD protect users from archive bombs.

Suggested defaults:

- warn above 50 MB compressed
- warn above 250 MB uncompressed
- reject if uncompressed size exceeds local policy
- reject extremely high compression ratios
- reject nested archives unless opened explicitly by user action

## Entry Point

The manifest `entry` field points to the first file loaded by the runtime.

Example:

```json
{
  "entry": "content/index.html"
}
```

The entry path MUST:

- refer to a file inside the archive
- pass all path rules
- not be remote
- not be a `file://`, `http://`, or `https://` URL

## Content Types

Runtimes SHOULD infer content types from file extensions or internal metadata.

Recommended mappings:

- `.html`: `text/html`
- `.css`: `text/css`
- `.js`: `text/javascript`
- `.json`: `application/json`
- `.svg`: `image/svg+xml`
- `.png`: `image/png`
- `.jpg` or `.jpeg`: `image/jpeg`
- `.webp`: `image/webp`
- `.wasm`: `application/wasm`

## Receipts Folder

The `receipts/` folder is reserved for exported receipts bundled by an author or generated by a runtime during export.

A runtime MUST NOT write into the original capsule archive just because the capsule ran. Runtime receipts should live in the runtime's local data store unless the user explicitly exports them.
